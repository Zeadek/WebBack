Задание 1

Подключение к учебному серверу по SSH с помощью клиента Putty

Устанавливаем программу PuTTY, вводим адрес сервера(kubsu-dev.ru), порт(58528), логинимся
  1. ping kubsu.ru
    2. Команда ping позволяет отправить серию небольших пакетов данных на указанное устройство и показать время ответа Она используется для устранения неполадок подключения.
  2. nslookup A-записи kubsu.ru
    3. nslookup — это инструмент сетевого администрирования для запроса cистемы доменных имён. На скриншоте отображен полученный адрес kubsu.ru
  3. nslookup A-записи kubsu-dev.ru
    4. На скриншоте отображен полученный адрес kubsu-dev.ru
  4. nslookup MX-записи kubsu.ru
    5. MX-запись (Mail exchange) — это запись, которая отвечает за сервер, через который будет работать почта На скриншоте отображены полученный адреса почт kubsu.ru
  5. nslookup MX-записи kubsu-dev.ru
    6. На скриншоте отображено, что у kubsu-dev.ru нет серверов, отвечающих за почту
  6. whois kubsu.ru
    7. whois — это команда для получения информации о доменах или IP-адресах На скриншоте отображены полученные дата регистрации домена kubsu.ru, данные владельца, днс-сервера
  7. whois kubsu-dev.ru
    8. На скриншоте отображены полученные дата регистрации домена kubsu-dev.ru, данные владельца, днс-сервера

Клонирование репозитория

Переходим в директорию www при помощи команды cd, туда клонируем репозиторий из github

Скачивание файлов с удаленного сервера при помощи FileZilla

С помощью программы FileZilla соединяемся с учебным сервером по протоколу SFTP и копируем на локальный компьютер файлы задания из каталога www

Добавление изменений на git и на сайт

Обновляем репозиторий на github, далее подключаемся к учебному серверу по SSH с помощью клиента Putty, переходим в директорию репозитория(cd www/WebBack), и обновляем при помощи команды git pull
